# Claude Instructions

Use `README.md` and `project.md` as source of truth.

## This is a hackathon. Optimize for:
- working end-to-end demo
- speed
- clarity

## Priorities
1. Working MVP with fixture data
2. Valid outputs (RDKit-checked SMILES)
3. Clean website demo
4. Polish after pipeline works
5. Advanced modeling only if time allows

## Rules
- Prefer simple working baselines over advanced architectures
- Do not overengineer
- No auth, no database, no production infra
- Demo mode (fixtures) ships first; live model is a bonus
- Keep code modular, readable, easy to debug
